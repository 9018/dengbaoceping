# 测评辅助平台阶段总结（截至当前）

## 1. 项目目标（当前口径）
- 输入：指导书、国标/行标、历史测评记录（Excel）、拓扑/设备截图、交换机与 Linux 配置。
- 输出：可提交的“测评记录 Excel”（不是报告）。
- 结构：重点使用二级、三级指标。
- 能力：按你的历史记录风格生成，并支持扩展指标按领域自动补充。

## 2. 当前进度概览
- 基础环境：已完成（核心服务可用）。
- 数据接入：已完成首批样本接入与结构化。
- 指标模型：已完成二/三级模型与扩展类型基础框架。
- 平台页面：已完成上传、文件管理、预览展示（持续优化中）。
- 自动生成：已完成前置数据链路，生成器最终版仍在收口阶段。

## 3. 已完成事项
### 3.1 基础部署
- 已在测试机创建项目目录：`/opt/cpgj`。
- 已部署并运行核心服务：
  - Elasticsearch（知识库核心）
  - Kibana（检索/核验）
  - Web 上传与预览平台（Docker）
- 已完成基础索引初始化与单机参数调整。

### 3.2 数据入库与结构化
- 已接收并处理你的首批样本（指导书、标准、历史记录、拓扑/截图、配置）。
- 已完成历史 Excel 结构化抽取，形成可检索的二/三级指标数据。
- 已完成指导书文本切分入库（规则片段化）。
- 已完成配置类证据文本入库。

### 3.3 指标与模板模型
- 已建立二/三级指标目录（indicator catalog）。
- 已建立运行时模板（record template）用于生成前的补充映射。
- 已建立扩展类型基础机制（当前已有通用/云，已预留多扩展框架）。

### 3.4 平台功能
- 已完成上传页面。
- 已完成已上传文件列表。
- 已完成文件删除功能。
- 已完成“模板补充预览”和“指标明细”页面能力。
- 已打通外部访问端口（NPS 映射）。

## 4. 当前主要问题与已识别风险
- 展示层仍需按你的记录习惯进一步收敛（领域分组顺序、二级项顺序、列定义）。
- 标准 PDF 自动抽取受运行环境限制，部分规则仍需通过“样本 + 规则词典”增强校准。
- 扩展类型目前以现有样本命中为主，仍需补齐“移动互联/物联网/工控”等扩展条目。

## 5. 下一步计划（按优先级）
### P0（先完成）
1. 固化“按领域分表”展示口径，完全贴合你的测评记录阅读习惯。
2. 固化二级项顺序（例如计算环境先“身份鉴别”，再“访问控制”）。
3. 统一“数量列”口径（按你指定的列位置和含义展示）。

### P1（核心能力）
1. 完成多扩展类型全量落库（通用、云计算、移动互联、物联网、工控等）。
2. 完成“按领域->二级->三级”自动补充引擎稳定版。
3. 输出第一版可提交 Excel 记录（保持你的模板与排版习惯）。

### P2（质量提升）
1. 引入生成前质量闸门（缺项、错位、重复、证据缺失）。
2. 引入人工修改回流，持续优化风格贴合度。
3. 形成可追溯链路（三级项 -> 证据 -> 生成文本）。

## 6. 项目整体大纲（最终形态）
1. 数据层：标准/指导书/历史记录/证据统一入库。
2. 模型层：二/三级指标目录、扩展类型目录、风格画像。
3. 引擎层：
   - 领域归位引擎
   - 扩展补充引擎
   - 生成引擎（Excel）
   - 质量闸门引擎
4. 平台层：上传管理、预览核验、生成导出、审计追踪。
