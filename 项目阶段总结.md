## 3. 本阶段关键修正

### 3.1 修复了前端导出下载地址写死问题

浏览器实操时发现，前端导出下载 URL 仍然写死为 `http://127.0.0.1:8000/api/v1`，这会导致 Docker / Nginx 交付形态下下载链路失效。

已完成修正：

- `frontend/src/api/http.ts` 导出 `apiBaseUrl`
- `frontend/src/api/exports.ts` 改为基于 `apiBaseUrl` 拼接下载地址

修正后，本地 Vite 与 Docker 反代两种运行形态共用同一套前端逻辑。

### 3.2 修复了前端容器构建错误继承本地开发 API 地址的问题

浏览器实操时还发现，前端 Docker 构建会把本地开发用的 `VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1` 打进静态包，导致容器部署后页面请求直接打向浏览器本机 `8000`，出现 `Network Error`。

已完成修正：

- `frontend.Dockerfile` 增加：
  - `ARG VITE_API_BASE_URL=/api/v1`
  - `ENV VITE_API_BASE_URL=${VITE_API_BASE_URL}`

修正后，前端容器默认通过 Nginx `/api/` 反向代理访问后端，Docker 交付链路恢复正常。
