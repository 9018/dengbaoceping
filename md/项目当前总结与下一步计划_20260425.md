# 项目当前总结与下一步计划（2026-04-26）

## 一、当前项目状态

项目已经从“等级保护测评主链路工具”升级为“主链路 + 知识库 + 历史样本 + 匹配解释 + 交付导出”的一体化测评工作台。当前核心链路为：项目管理、证据上传、OCR、字段抽取、测试对象匹配、指导书条目匹配、测评记录生成、复核审批、TXT/Excel 导出与下载。

## 二、已完成能力

### 1. 测评主链路

- 项目创建、列表、详情与状态管理。
- 证据上传、归档、OCR 识别、字段抽取与字段复核。
- 基于抽取字段生成测评记录，支持复核、审批、导出状态流转。
- 记录导出前会校验全部记录是否已审批或已导出，避免未闭环记录进入交付文件。

### 2. OCR 与字段抽取

- OCR provider 支持 `mock`、`paddle`、`real` 占位。
- PaddleOCR 接入已补齐运行依赖、启动配置和失败结果持久化。
- OCR 结果统一沉淀到证据文本与结构化结果中，下游字段抽取继续复用同一入口。

### 3. 测试对象匹配

- 证据支持匹配测试对象资产，并持久化匹配分数、匹配原因、匹配状态。
- 支持在复核页查看建议资产、匹配分数和原因，并人工确认绑定。
- 资产模型已补充测试对象类型、主 IP 等字段，便于后续按真实设备组织证据和导出。

### 4. 指导书知识库

- 固定读取 `md/指导书.md`，支持 Markdown 章节导入。
- 支持章节列表、详情、关键词搜索。
- 指导书条目沉淀 `guidance_code`、章节路径、标题、正文、关键词、核查要点、证据要求和记录建议。

### 5. 历史人工测评记录库

- 支持导入人工三级测评 Excel 历史样本。
- 支持按 sheet、控制点、符合情况、资产类型筛选和关键词检索。
- 历史记录保留编号、扩展标准、控制点、测评项、结果记录、符合情况、分值和关键词。

### 6. 指导书与历史样本关联

- 新增 guidance-history 关联层，支持按指导书条目计算相似历史样本。
- 支持从指导书查看关联历史记录，也支持从历史记录反查指导书条目。
- 关联结果包含匹配分数与命中原因，为后续记录生成增强和导出补全提供依据。

### 7. 测评记录 Excel 导出

- 新增 Excel 导出入口：`POST /api/v1/projects/{project_id}/export-excel`。
- 下载入口继续复用：`GET /api/v1/exports/{export_id}/download`。
- TXT 导出入口保持不变：`POST /api/v1/projects/{project_id}/export`。
- Excel 文件使用 `openpyxl` 生成，不依赖 LibreOffice。
- 支持两种模式：
  - `official`：只输出正式交付列。
  - `debug`：在正式列后追加测试对象、证据文件、指导书依据、匹配分数、历史样本 ID。
- Excel 按资产分 sheet，sheet 名优先使用已确认测试对象名称，缺失时回退记录资产或 evidence.device。
- 表头统一样式、冻结首行、自动列宽，符合情况按历史样本文字输出。

### 8. 前端导出中心

- 导出中心保留 TXT 导出按钮。
- 新增 Excel 导出按钮。
- 新增 official/debug 模式选择。
- 导出任务历史展示文件名、格式、模式、状态、记录数、文件大小、更新时间和下载入口。

## 三、关键技术变更

### 后端

- 新增/扩展资产匹配、指导书匹配、历史样本关联和 Excel 导出服务。
- `ExportService` 继续作为导出编排层，统一处理记录状态校验、文件落盘、ExportJob 入库与下载响应。
- `ExcelExportService` 独立负责 workbook 数据装配、sheet 分组、样式、列宽和 official/debug 列切换。
- `ExportJob` 增加 `mode` 字段，用于记录 Excel 导出模式。
- 新增 Alembic 迁移覆盖资产匹配、指导书匹配、导出模式等数据库结构变更。

### 前端

- 复核页接入测试对象匹配和指导书匹配确认能力。
- 类型定义补齐资产匹配、指导书匹配、导出模式等字段。
- 导出中心接入 Excel 导出 API 与模式选择。

## 五、当前风险与注意事项

1. 工作区存在大量运行时缓存、pycache、ES data、快照与上传文件变更，这些不属于本次业务代码交付，不应纳入提交。
2. 当前 Excel 导出依赖“证据已确认测试对象与指导书”的数据质量；未确认时会按回退规则导出，但 debug 信息会变少。
3. guidance-history 关联仍是规则匹配第一版，适合当前验证阶段；如果历史样本规模扩大，需要考虑批量重算、异步任务或更强检索能力。
4. 浏览器端真实点击导出链路已经具备启动条件，但最终交付仍建议在目标运行环境中再做一次人工黄金路径验收。

## 六、下一步计划

### P0：交付稳定性

1. 清理提交范围，只提交业务代码、迁移、测试、前端和本文档。

2. 创建项目、上传证据、执行 OCR/字段抽取、确认测试对象与指导书。
3. 生成记录并完成复核审批。
4. 在导出中心分别导出 TXT、Excel official、Excel debug。
5. 下载 xlsx，校验 sheet、表头、冻结首行、历史样本字段和 debug 附加列。

1. 将 confirmed guidance 与 top history sample 作为记录生成参考输入。
2. 优化 guidance-history 关联解释，让复核人员能看懂为什么命中。
3. 评估是否需要引入批量重算入口或后台任务。
4. 根据真实样本继续补充规则、模板和字段抽取测试。

## 七、结论

当前项目已经具备可演示、可测试、可交付的核心闭环。下一阶段的抓手不是继续堆功能，而是稳定提交范围、验证主链路、用真实样本打磨匹配和导出质量。先把交付链路做稳，再向自动化生成质量升级。
